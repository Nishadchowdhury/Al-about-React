How to work react

DOM and Vartual DOM যেমন একটা বই এর ক্ষেত্রে সেটার কোন একটা অংশ কে যদি কখনো পরিবর্তন করতে চাই তবে সেটার আগের কপি টার সাথে তুলনা করে কাজটা করতে হবে
DOM এর মধ্যেও কিছু পরিবর্তন করতে হলে সেটাকে আবার re render করতে হবে ।
যেটা অনেক বেশি expensive অর্থাৎ এই ক্ষেত্রে অনেক বেশি power and টাইম খরচ হয় ।

যেটা আমারা ui তে দেখি সেটা হল orginal DOM , react আর একটা dom বানিয়ে নেয় যেটা দেখতে Originel DOM এর মতই দেখতে কিন্তু কিছুটা আলাদা কিছুটা simple looking
এটার সুবিধা হল js DOM এর মধ্যে গিয়ে খুব দ্রুত গিয়ে চেক করে ফেলতে পারে । যে কোথায় কি কি পরিবর্তন হয়েছে
তারপর সে actual dom and vartual এর মধ্যে তুলনা করে সহজেই শুধু মাত্র সেই state টাকে ui তে আপডেট করে ।
এটা করে Diff Algrorith এর সাহায্যে ।
এটা কাজ করে react fiber architecture algorithm এর উপর
এই react fiber algorithm এর কাজ হল সে খুঁজে খুঁজে বের করে যে কিভাবে আপডেট করলে আপডেট টা Efficiant হয়
আর সে কাজ করে Asynchronous way তে
এতে করে কখনো এমন হতে পারে যে কোন একটা part আপডেট করতে পারতেছেনা তখন যেহেতু পুরা পেজটা আটকে না যায় সে জন্য এটা Asynchronously কাজ করে
যেমন আরেকটা জিনিস আছে যেটা Performance এর অনেক ভাল করে দেয়
সেটা হল কোন array of data কে loop করলে সেখানে বলে দেয় যে একটা key দেয়ার জন্য , যাতে করে react বুজতে পারে যে কোন state এর ডাটা কে replace করে update করতে হবে ।

মুল কথাঃ- react memory এর মধ্যে real dom এর কাছাকাছি একটা vartual dom তৈরি করে যেটা যাতে তুলনা করতে পারে যে ui এর কোন অংশে কি কি পরিবর্তন এসেছে ।
তারপর সেই অনুসারে তারা ui কে change করবে
এটা করার জন্য diff algorithm ব্যেবহার করে
